【移行記事】migrationファイルの生成・記述方法のまとめ【随時更新】

2019/11/26
Tech Memo

17:23:38

Rails における migration を使ったテーブル定義の変更方法は、Qiita を筆頭に、ちょっとググればすぐ出てきます。大変ありがたいのですが、一向に migration ファイルの書き方を覚えられないし、毎回ググるのも面倒になってきたので、「自分のブログを見れば全部わかる」状態を目指して、まとめることにしました。

まとめといっても、実際に自分が使ったことのあるものだけを記載しています。随時更新していきます。

<h2 id=“既存テーブルに対する変更”>既存テーブルに対する変更

<h3 id=“カラムを追加する”>カラムを追加する

ファイル生成コマンド<br><code>rails g migration AddColumnTo[クラス名] [カラム名]:[データ型]</code> <br>※追加するカラムはスペース区切りで複数指定可

<pre class=“code lang-ruby” data-lang=“ruby” data-unlink><span class=“synComment”># migration ファイル<span class=“synPreProc”>def <span class=“synIdentifier”>change <span class=“synComment”># add_column :対象テーブル名, :対象カラム名, :型 add_column <span class=“synConstant”>:users, <span class=“synConstant”>:age, <span class=“synConstant”>:integer<span class=“synPreProc”>end</pre>

<h3 id=“カラム名を変更する”>カラム名を変更する

ファイル生成コマンド<br><code>rails g migration Rename[変更前のカラム名]ColumnTo[変更後のカラム名]</code>

<pre class=“code lang-ruby” data-lang=“ruby” data-unlink><span class=“synComment”># migration ファイル<span class=“synPreProc”>def <span class=“synIdentifier”>change <span class=“synComment”># rename_column :[対象テーブル名], :[変更前のカラム名], :[変更後のカラム名] rename_column <span class=“synConstant”>:users, <span class=“synConstant”>:uid, <span class=“synConstant”>:user_id<span class=“synPreProc”>end</pre>

<h3 id=“カラムのデータ型を変更する”>カラムのデータ型を変更する

ファイル生成コマンド<br><code>rails g migration ChangeDatatype[カラム名]Of[テーブル名]</code>

<pre class=“code lang-ruby” data-lang=“ruby” data-unlink><span class=“synComment”># migration ファイル<span class=“synPreProc”>def <span class=“synIdentifier”>change <span class=“synComment”># change_column :対象テーブル名, :対象カラム名, :変更後のデータ型 change_column <span class=“synConstant”>:users, <span class=“synConstant”>:name, <span class=“synConstant”>:string<span class=“synPreProc”>end</pre>

※ただし、PostgreSQL で integer や boolean 型に変更すると<code>PG::DatatypeMismatch: ERROR</code>というエラーが起きる場合があります。<code>change_column</code>メソッドを以下のようにすると解消します。

<pre class=“code lang-ruby” data-lang=“ruby” data-unlink>change_column <span class=“synConstant”>:users, <span class=“synConstant”>:age, <span class=“synSpecial”>'<span class=“synConstant”>integer USING CAST(age AS integer)<span class=“synSpecial”>'</pre>

<h3 id=“カラムにオプションを設定する”>カラムにオプションを設定する

<h4 id=“デフォルト値”>デフォルト値</h4>

ファイル生成コマンド<br><code>rails g migration AddDefalutValue[カラム名]Of[テーブル名]</code>

<pre class=“code lang-ruby” data-lang=“ruby” data-unlink><span class=“synComment”># migration ファイル<span class=“synPreProc”>def <span class=“synIdentifier”>change <span class=“synComment”># change_column :対象テーブル名, :対象カラム名, :型, default: デフォルト値 change_column <span class=“synConstant”>:users, <span class=“synConstant”>:age, <span class=“synConstant”>:integer, <span class=“synConstant”>default: <span class=“synConstant”>20<span class=“synPreProc”>end</pre>

<h4 id=“外部キー制約”>外部キー制約</h4>

ファイル生成コマンド<br><code>rails g migration AddForeignKeyTo[テーブル名] [カラム名]:references</code>

<pre class=“code lang-ruby” data-lang=“ruby” data-unlink><span class=“synComment”># migration ファイル<span class=“synPreProc”>def <span class=“synIdentifier”>change <span class=“synComment”># add_reference :対象テーブル名, :対象カラム名, foreign_key: true add_reference <span class=“synConstant”>:users, <span class=“synConstant”>:club, <span class=“synConstant”>foreign_key: <span class=“synConstant”>true<span class=“synPreProc”>end</pre>

<h4 id=“NOT-NULL 制約”>NOT NULL 制約</h4>

ファイル生成コマンド<br><code>rails g migration Change[カラム名]Of[テーブル名]</code>

※<code>false</code>を<code>true</code>とすれば制約を外すことができる。

<pre class=“code lang-ruby” data-lang=“ruby” data-unlink><span class=“synComment”># migration ファイル<span class=“synPreProc”>def <span class=“synIdentifier”>change <span class=“synComment”># change_column_null :対象テーブル名, :対象カラム名, false change_column_null ::users, <span class=“synConstant”>:user_id, <span class=“synConstant”>false<span class=“synPreProc”>end</pre>
