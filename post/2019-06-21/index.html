【移行記事】『Everyday Rails - RSpecによるRailsテスト入門』を読んだ

2019/06/21
Tech Memo

12:36:45

読み終わってから随分時間が経ってしまいましたが、<a href=“https://leanpub.com/everydayrailsrspec-jp">『Everyday Rails - RSpec による Rails テスト入門』</a>を読みました。

本書は Ruby のテスティングフレームワークである「RSpec」の解説書です。

販売形態は電子書籍のみですが、Ruby や Rails のバージョンアップに伴って本書の内容も無料でアップデートされています。

感想

Ruby と Rails の知識はある程度必要

本書では Ruby や Rails に関する詳細な解説はありません。前提知識がないと理解が追いつかないと思うので、ある程度 Ruby、Rails の知識をインプットしておいた方が良いです。

といっても RSpec を勉強しようという人は Ruby や Rails を知っている人がほとんどだと思いますが。

RSpec の知識はほとんどいらない

逆に RSpec の知識はほとんど必要ありません。

僕自身、RSpec の知識はほぼゼロの状態で読み始めました。読み終えてから RSpec でテストコードをバリバリ書けています。

手を動かすことで頭に入ってくる

本書では事前に用意されている Rails のサンプルアプリケーションにテストコードを追記していくチュートリアル形式で学習を進めていきます。

もちろん手を動かさずに読むだけでも良いのですが、手を動かした方が理解が進みやすかったです。というか、僕は読んでるだけだと集中力が続かない系男子なのでチュートリアル形式がありがたかったです。

章ごとにサンプルアプリケーションのソースコートが用意されているので途中の章から始めることもできます。

RSpec のセットアップから解説されていて分かりやすい

サンプルアプリケーションに対して RSpec をセットアップ方法からスペック（テストコード）の書き方、テストの実行方法まで丁寧に解説されていて非常にわかりやすかったです。

他にもファクトリを利用したテストデータを作成する方法や、DRY にするテクニックなども紹介されていて、とりあえず本書を読めば RSpec の基本は一通り押さえられる印象です。

まとめ

本書を読めば、RSpec を使った基本的なテストは独力で行えるようになります。

僕は RSpec どころか、テストコードを書く環境に身を置いたことがない人間ですが、本書のおかげでテストコードが書けるようになりました。（個人開発でテストコードを書いています）

本書に興味を持った方は、こちらの記事に Q&amp;A が掲載されているので購入の参考になります。（購入方法についても触れられています）
