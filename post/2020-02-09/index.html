【移行記事】【Rails】『omniauth-twitter』でログイン機能を実装する手順

2020/02/09
Dialy

21:58:35

いちいち思い出すのが面倒なので手順を記録しておく。なお『devise』と連携して実装する場合はこの通りでは無い。

動作確認環境・Rails 6.0.0・Ruby 2.6.3・omniauth 1.9.0・omniauth-twitter 1.4.0

１．TwitterAPI の登録 CallBack_URL の設定に注意。適切に設定しないとエラーになる。ローカル環境で開発する場合はとりあえず以下にしておけば OK。（ポート番号は環境に合わせて。）

http://127.0.0.1:3000/auth/twitter/callback http://localhost:3000/auth/twitter/callback ２．Gem のインストール

以下の 3 つの gem をインストール。bundle install を忘れずに。

<pre class=“code” data-lang="" data-unlink># Gemfile

gem &#39;omniauth&#39;gem &#39;omniauth-twitter&#39;gem &#39;dotenv-rails&#39;</pre>

３．API キーの登録<code>.env</code>に API キーを設定する。いわずもがな、<code>.env</code>は git 管理対象から外しておく。

<pre class=“code” data-lang="" data-unlink># .env

TWITTER_KEY=&#39;&lt;KEY&gt;&#39;TWITTER_SECRET=&#39;&lt;SECRET&gt;&#39;</pre>

ommiauth にもキーを設定する。<code>config/initializers/omniauth.rb</code>を作成し以下のように記述。

<pre class=“code lang-ruby” data-lang=“ruby” data-unlink><span class=“synComment”># config/initializers/omniauth.rb

<span class=“synType”>Rails.application.config.middleware.use <span class=“synType”>OmniAuth::<span class=“synType”>Builder <span class=“synStatement”>do provider <span class=“synConstant”>:twitter, <span class=“synIdentifier”>ENV[<span class=“synSpecial”>&quot;<span class=“synConstant”>TWITTER_KEY<span class=“synSpecial”>&quot;], <span class=“synIdentifier”>ENV[<span class=“synSpecial”>&quot;<span class=“synConstant”>TWITTER_SECRET<span class=“synSpecial”>&quot;]

<span class=“synType”>OmniAuth.config.on_failure = <span class=“synType”>Proc.new { |<span class=“synIdentifier”>env| <span class=“synType”>OmniAuth::<span class=“synType”>FailureEndpoint.new(env).redirect_to_failure }<span class=“synStatement”>end</pre>

４．User モデル作成 User モデルを作成し、取得した情報をセットするクラスメソッドを定義する。

取得できる情報は<a href=“https://github.com/arunagw/omniauth-twitter#authentication-hash">こちら</a>に記載があるので、必要に応じて取得するようにする。

<code>rails db:migrate</code>を忘れずに実行する。

<pre class=“code” data-lang=”" data-unlink>rails g model user provider:string uid:string user_name:string screen_name:string image_url:string</pre>

<pre class=“code” data-lang="" data-unlink># マイグレーションファイル

<pre class=“code lang-ruby” data-lang=“ruby” data-unlink><span class=“synComment”># app/model/user.rb

<span class=“synPreProc”>class <span class=“synType”>User &lt; <span class=“synType”>ApplicationRecord <span class=“synPreProc”>def <span class=“synConstant”>self.<span class=“synIdentifier”>find_or_create_from_auth(auth) provider = auth[<span class=“synConstant”>:provider] screen_name = auth[<span class=“synConstant”>:extra][<span class=“synconstant”>:raw_info][<span class=“synConstant”>:screen_name] uid = auth[<span class=“synConstant”>:uid] user_name = auth[<span class=“synConstant”>:info][<span class=“synconstant”>:name] image_url = auth[<span class=“synConstant”>:info][<span class=“synconstant”>:image].gsub(<span class=“synSpecial”>/<span class=“synConstant”>_normal<span class=“synSpecial”>.<span class=“synConstant”>jpg<span class=“synSpecial”>/, <span class=“synSpecial”>&quot;<span class=“synConstant”>.jpg<span class=“synSpecial”>&quot;)

<span class=“synConstant”>self.find_or_create_by(<span class=“synConstant”>provider: provider, <span class=“synConstant”>uid: uid) <span class=“synStatement”>do |<span class=“synIdentifier”>user| user.user_name = user_name user.image_url = image_url user.screen_name = screen_name <span class=“synStatement”>end <span class=“synPreProc”>end<span class=“synPreProc”>end</pre>

５．Session コントローラー作成ログイン・ログアウトを行うコントローラーを作成する。合わせてルーティングも定義する。

６．ヘルパーメソッドを定義ログイン中のユーザー情報の取得および、ログイン中か確認するヘルパーメソッドを<code>application_controller.rb</code>定義し各所で呼び出せるようにしておく。

７．動作確認以上で準備完了。<code>get ‘/auth/twitter’</code>でログイン、<code>get ‘/signout’</code>でログアウトできる。
