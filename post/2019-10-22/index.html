<h2 id=“はじめに”>はじめに

つぎにつくる予定の Web アプリでは、認証（ログイン）機能を実装しようと思っています。そこで認証機能を実装するための gem で有名な『devise』の最低限の動作確認をしてみました。

環境は以下の通りです 💡

・Ruby 2.6.3・Rails 6.0.0・devise 4.7.1 （現時点の最新バージョン）

<h2 id=“実装”>実装

<h3 id=“１ rails-new する”>１．rails new する

<code>Rails6.0.0</code>でアプリを新規作成します。

<pre class=“code” data-lang="" data-unlink>rails 6.0.0 new gem-devise-sample</pre>

トップページが表示されることを確認します。

<h3 id=“２ devise-をインストールする”>２．devise をインストールする

Gemfile に以下を記述し、<code>bundle install</code>。

<pre class=“code lang-ruby” data-lang=“ruby” data-unlink><span class=“synComment”># Gemfile

gem <span class=“synSpecial”>'<span class=“synConstant”>devise<span class=“synSpecial”>'</pre>

続けて、以下のコマンドで devise の設定ファイルを作成します。

<pre class=“code” data-lang="" data-unlink>$ rails g devise:install</pre>

<h3 id=“３ devise 対応の User モデルを生成する”>３．devise 対応の User モデルを生成する

User モデルをつくります。以下のコマンドで devise 対応の User モデルを生成。

<pre class=“code” data-lang="" data-unlink>$ rails g devise User</pre>

以下のマイグレーションファイルが生成されるので、<code>rails db:migrate</code>でマイグレーションを実行。コメントアウトされているカラムがあるが、今回は最低限の動作確認のみのため、そのままにしておきます。

<pre class=“code” data-lang="" data-unlink># frozen_string_literal: true

class DeviseCreateUsers &lt; ActiveRecord::Migration[6.0] def change create_table :users do |t| ## Database authenticatable t.string :email, null: false, default: &#34;&#34; t.string :encrypted_password, null: false, default: &#34;&#34;

この時点で、<a href=“http://localhost:3000/users/sign_in”>http://localhost:3000/users/sign_in</a>を開くと最低限のログインページができています。

しかし、この状態だと認証後のリダイレクト先ページが Rails のデフォルトページ（Yay! You’re on Rails!の画面）になっているので、リダイレクト先ページをユーザー個人のプロフィールページに変更します。

<h3 id=“４トップページをつくる”>４．トップページをつくる

Pages コントローラーを index、show アクションつきで生成。

<pre class=“code” data-lang="" data-unlink>$ rails g controller Pages index show</pre>

ルーティングを設定をします。root を先ほど生成した<code>pages#index</code>に。なお、<code>devise_for :users</code>は devise 対応の User モデルを生成した時点で自動で設定されます。

<pre class=“code lang-ruby” data-lang=“ruby” data-unlink><span class=“synComment”># route.rb

<span class=“synType”>Rails.application.routes.draw <span class=“synStatement”>do root <span class=“synSpecial”>'<span class=“synConstant”>pages#index<span class=“synSpecial”>' get <span class=“synSpecial”>'<span class=“synConstant”>pages/show<span class=“synSpecial”>' devise_for <span class=“synConstant”>:users<span class=“synStatement”>end</pre>

トップページのビューを編集します。今回は、ログイン状態によって、表示するリンクが切り替わるようにしたいと思います。<code>user_signed_in?</code>でログイン状態を判定し、表示するリンクを切り替えます。

routes.rb に設定された<code>devise_for :users</code>により、‘new_user_session_path‘や‘destroy_user_session_path‘など、devise 用の名前付きルートが使えるようになっているので、その名前付きルートを使ってリンクを記述します。

<pre class=“code lang-html” data-lang=“html” data-unlink><span class=“synComment”>&lt;!– index.html.erb –&gt;

<span class=“synIdentifier”>&lt;<span class=“synStatement”>h1<span class=“synIdentifier”>&gt;トップページ<span class=“synIdentifier”>&lt;/<span class=“synStatement”>h1<span class=“synIdentifier”>&gt;<span class=“synIdentifier”>&lt;<span class=“synStatement”>p<span class=“synIdentifier”>&gt;Find me in app/views/pages/index.html.erb<span class=“synIdentifier”>&lt;/<span class=“synStatement”>p<span class=“synIdentifier”>&gt;<span class=“synIdentifier”>&lt;% if user_signed_in? %&gt; <span class=“synIdentifier”>&lt;%=<span class=“synConstant”> link_to<span class=“synIdentifier”> <span class=“synConstant”>‘ログアウト’<span class=“synIdentifier”>, destroy_user_session_path, <span class=“synType”>method<span class=“synIdentifier”>: :delete %&gt;<span class=“synIdentifier”>&lt;% else %&gt; <span class=“synIdentifier”>&lt;%=<span class=“synConstant”> link_to<span class=“synIdentifier”> <span class=“synConstant”>‘サインアップ’<span class=“synIdentifier”>, new_user_registration_path %&gt; <span class=“synIdentifier”>&lt;%=<span class=“synConstant”> link_to<span class=“synIdentifier”> <span class=“synConstant”>‘ログイン’<span class=“synIdentifier”>, new_user_session_path %&gt;<span class=“synIdentifier”>&lt;% end %&gt;</pre>

これで、ログイン状態によってリンクが切替わるようになりました。

<h3 id=“５認証後のリダイレクト先をユーザーページにする”>５．認証後のリダイレクト先をユーザーページにする

<code>Devise::Controllers::Helpers</code>の<code>after_sign_in_path_for</code>メソッドをオーバーライドすることで、認証後のリダイレクト先をカスタマイズできます。

<pre class=“code lang-ruby” data-lang=“ruby” data-unlink><span class=“synComment”># application.rb

<span class=“synPreProc”>def <span class=“synIdentifier”>after_sign_in_path_for(resource) <span class=“synSpecial”>&quot;<span class=“synConstant”>/user/<span class=“synSpecial”>#{current_user.id<span class=“synSpecial”>}&quot; <span class=“synPreProc”>end</pre>

※current_user で現在ログインしているユーザー情報を取得できる。

ユーザーページのビューを編集します。ログインユーザーのメールアドレスを表示するようにしてみました。

<pre class=“code lang-html” data-lang=“html” data-unlink><span class=“synComment”>&lt;!– show.html.erb –&gt;

<span class=“synIdentifier”>&lt;<span class=“synStatement”>h1<span class=“synIdentifier”>&gt;ユーザーページ<span class=“synIdentifier”>&lt;/<span class=“synStatement”>h1<span class=“synIdentifier”>&gt;<span class=“synIdentifier”>&lt;<span class=“synStatement”>div<span class=“synIdentifier”>&gt; あなたのメールアドレス：<span class=“synIdentifier”>&lt;%=<span class=“synConstant”> current_user.email<span class=“synIdentifier”> %&gt;<span class=“synIdentifier”>&lt;/<span class=“synStatement”>div<span class=“synIdentifier”>&gt;<span class=“synIdentifier”>&lt;<span class=“synStatement”>div<span class=“synIdentifier”>&gt; <span class=“synIdentifier”>&lt;%=<span class=“synConstant”> link_to<span class=“synIdentifier”> <span class=“synConstant”>‘戻る’<span class=“synIdentifier”>, root_path %&gt;<span class=“synIdentifier”>&lt;/<span class=“synStatement”>div<span class=“synIdentifier”>&gt;</pre>

さいごに、ユーザーページのルーティングを編集。

<pre class=“code” data-lang="" data-unlink># route.rb

Rails.application.routes.draw do root &#39;pages#index&#39; get &#34;user/:id&#34;, :to =&gt; &#34;pages#show&#34; # この 1 行を編集 devise_for :usersend</pre>

これで完成 😤

ログインするとそのユーザーのページが開きます。

<h2 id=“おわりに”>おわりに

ソースコードはこちらです 💁‍♂️

サインインやサインアップページのビューや、動作をカスタマイズするには<code>rails g devise:views users</code>や<code>rails g devise:controllers users</code>をして devise 対応のコントローラーやビューを生成する必要があるようです。今回は最低限の動作確認が目的のため、そこまでは手を出しませんでした。

サンプルアプリをつくるくらいなら簡単ですが、devise は多機能で、使いこなすのは一筋縄ではいかなそう、という印象を受けました。まあ、それでもゼロから認証機能をつくるよりはよっぽど簡単なのですが。
