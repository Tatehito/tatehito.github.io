【移行記事】【Ruby】『better_errors』を試す

2019/10/29
Tech Memo

08:00:00

Rails のエラー画面をリッチにする gem『<a href=“https://rubygems.org/gems/better_errors"&gt;better_errors&lt;/a>』を使ってみたので、導入方法を紹介します。

使用バージョン

・Ruby 2.6.3・Rails 6.0.0・better_errors 2.5.1（現行の最新版）・binding_of_caller 0.8.0（現行の最新版）

インストールする

Gemfile に<code>better_errors'</code>と<code>binding_of_caller'</code>を追記して<code>bundle install</code>。

開発用なので、development 環境のみにインストールします。

<pre class=“code lang-ruby” data-lang=“ruby” data-unlink><span class=“synComment”># Gemfile

group <span class=“synConstant”>:development <span class=“synStatement”>do gem <span class=“synSpecial”>'<span class=“synConstant”>better_errors<span class=“synSpecial”>', <span class=“synSpecial”>'<span class=“synConstant”>&gt;= 2.5.1<span class=“synSpecial”>' gem <span class=“synSpecial”>'<span class=“synConstant”>binding_of_caller<span class=“synSpecial”>', <span class=“synSpecial”>'<span class=“synConstant”>&gt;= 0.8.0<span class=“synSpecial”>'<span class=“synStatement”>end</pre>

エラーを発生させてみる

存在しないメソッドを呼ぶことで、意図的にエラーを発生させてみます。ちなみに、<code>Routing Error</code>では<code>better_errors</code>は動作せず、従来のエラー画面が表示されます。

<pre class=“code lang-ruby” data-lang=“ruby” data-unlink><span class=“synComment”># pages_controller.rb

<span class=“synPreProc”>class <span class=“synType”>PagesController &lt; <span class=“synType”>ApplicationController <span class=“synPreProc”>def <span class=“synIdentifier”>index local = <span class=“synConstant”>100 <span class=“synIdentifier”>@instance = <span class=“synSpecial”>'<span class=“synConstant”>hoge<span class=“synSpecial”>' sample <span class=“synPreProc”>end

<span class=“synPreProc”>def <span class=“synIdentifier”>sample undefined <span class=“synPreProc”>end<span class=“synPreProc”>end</pre>

<code>pages/index</code>にアクセスすると、<code>better_errors</code>のエラー画面が表示されました！

左半分に表示されたスタックトレースを選択すると、その時点での変数の状態などが右側に表示されます。

<code>index</code>が呼ばれたタイミングでは、ローカル変数<code>local</code>やインスタンス変数<code>@instance</code>が参照できています。

エラーが発生したタイミングは<code>sample</code>内なので、ローカル変数は定義されていないことがわかります。インスタンス変数<code>@instance</code>は変わらず参照できていることが分かります。

導入は<code>bundle install</code>するだけでめちゃくちゃ簡単なので、とりあえず入れておくとデバッグが捗るかもしれません！
